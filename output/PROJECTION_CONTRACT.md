1. PROJECTION DOCTRINE
Projection Doctrine

A Projection is a deterministic derived representation generated from authoritative system truth.

A Projection exists solely to transform authoritative historical facts into visibility structures consumable by read systems.

A Projection is never authoritative.

A Projection is never a business entity.

A Projection is never a source of truth.

A Projection exists only because authoritative truth already exists.

Authority creates facts.

Projections create visibility.

Truth survives projection loss.

Visibility does not.

Derived-State Doctrine

All Projection state is derived.

No Projection may contain independently owned truth.

All Projection state must be reproducible from authoritative system history.

Projection state is disposable.

Projection state is reconstructable.

Projection state is replaceable.

Event Transformation Doctrine

Events are authoritative historical facts.

Projections consume events.

Projections transform events into visibility.

Projections never alter event meaning.

Projections never reinterpret authority.

Projections never invent history.

Visibility Doctrine

Visibility exists through Projections.

Visibility never creates truth.

Visibility never modifies truth.

Visibility never replaces truth.

Visibility is a representation layer only.

Authority Separation Doctrine

Authority and Projection are permanently separated.

Authority owns:

Business Facts
Lifecycle Facts
Financial Facts
Infrastructure Facts
Historical Facts

Projection owns:

Representation
Visibility
Read Optimization
Consumption Readiness

Projection ownership never extends into authority ownership.

Reconstruction Doctrine

Every Projection must be reconstructable from authoritative history.

Projection destruction must never threaten business continuity.

Projection loss must never threaten system truth.

Replay-Safe Doctrine

Projection generation must produce identical visibility when replayed from identical historical truth.

Replay must never alter authority.

Replay must never create authority.

Replay must only regenerate visibility.
